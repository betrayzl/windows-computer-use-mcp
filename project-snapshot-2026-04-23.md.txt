一、项目来源与背景
项目名称：Windows Computer Use MCP
项目定位：构建一个高精度、高可靠性的 Windows 自动化 MCP (Model Context Protocol) 服务器，使 AI 智能体能够对 Windows 操作系统实现“像素级”的操控与感知。

核心驱动背景：

当前 AI 智能体与操作系统交互普遍受限于文本接口或脆弱的自动化脚本。

Windows 桌面自动化面临多屏 DPI 异构、窗口焦点劫持、UI 遮挡等实际环境问题。

需要一个具备“环境自愈”能力的执行引擎，而非简单的坐标转发器。

项目目标：

视觉与操作一致性：确保 AI 智能体通过截图看到的“逻辑坐标”与鼠标点击的“物理坐标”严格对应。

环境感知型执行：自动处理窗口遮挡、焦点抢夺失败等异常场景。

工业级稳定性：为上层 AI 应用提供可靠、可扩展的计算操作能力。

二、核心交付物定义（回顾）
项目第一阶段的交付物已经明确并实现：

交付物	说明
高性能截图引擎	基于 DXGI 桌面复制 API，可排除指定进程窗口（如自身控制台），输出 Base64 格式 JPEG。
精准输入控制	支持鼠标移动、点击、拖拽、滚轮；支持文本输入；支持元素级键盘模拟。

三、第一阶段已解决的问题
截至 2026-04-21 项目快照，以下关键技术障碍已被攻克：

HWND 句柄截断问题

问题：JavaScript Number 类型无法安全容纳 64 位窗口句柄，导致窗口操作失败。

解决方案：Rust 层统一使用 i64 传递句柄，TS 层透传，彻底解决 64 位兼容性问题。

截图窗口状态破坏

问题：为清理截图画面而临时隐藏窗口后，若程序崩溃则窗口永久隐藏。

解决方案：Rust 实现 WindowRestoreGuard RAII 模式，自动在作用域结束时恢复窗口显示。

多显示器 DPI 缩放错误

问题：界面元素在 4K 屏与 1080P 屏混合时点击位置偏移。

解决方案：实现 logicalToPhysical 函数，根据目标显示器实时 DPI 和物理偏移计算绝对坐标。

窗口焦点劫持防御

问题：SetForegroundWindow 在后台进程中被 Windows 拦截，窗口仅任务栏闪烁而无法激活。

解决方案：实现三级激活回退机制，并在关键时刻模拟物理 Alt+Tab （见 window.rs）。

基础编排工具就绪

node build.js 流程打通，原生模块可通过 native-loader.ts 在开发环境与打包环境下正确加载。

四、当前阶段的核心问题与解决方案
4.1 问题描述（阻断级缺陷）
现象：
运行 test_driver.js 中的 Notepad Lifecycle Closed-Loop 场景时：

可以正常启动记事本 (open_app)。

可以成功将记事本窗口聚焦并键入文本 (type)。

无法关闭窗口：执行 key("alt+f4") 后，窗口无反应，最终测试超时失败。

根本原因：
Rust 原生模块 input.rs 中的键盘事件处理器无法解析组合键字符串。

executor.ts 调用 inputController.key("alt+f4", "press")。

Rust 端 map_key 函数采用简单的单键 match 表，将整个 "alt+f4" 作为整体匹配，结果命中 _ => Err() 分支。

Rust 错误被 MCP 协议转换为 JSON-RPC 错误响应，但测试驱动未检查工具返回状态，导致测试继续轮询窗口消失并最终超时。

影响范围：
所有依赖快捷键的自动化操作（关闭窗口、复制 ctrl+c、粘贴 ctrl+v、打开任务管理器 ctrl+shift+esc 等）全部失效。

4.2 详细解决方案（已最终确认）
采用 Assistant 架构分析报告 中提议的 Rust 层完整重构方案，其相比于简单热修复具有更好的工程扩展性。

方案目标
使 InputController::key 方法能够解析并执行以 + 分隔的组合键序列。

正确处理修饰键（Alt, Ctrl, Shift, Meta）与普通键的按键顺序。

保持与现有 TypeScript 层调用的兼容性，减少上层改动。

实施步骤
第一步：重构 native/src/input.rs

删除原实现：

移除现有 key 方法实现。

移除原 map_key 函数。

替换为以下实现（关键代码已在实际环境中验证可行性）：

rust
// === 新增：组合键解析与执行逻辑 ===

impl InputController {
    /// 解析组合键序列，返回修饰键列表与主键
    fn parse_key_sequence(&self, sequence: &str) -> Result<(Vec<enigo::Key>, enigo::Key)> {
        let tokens: Vec<&str> = sequence.split('+').map(|s| s.trim()).collect();
        if tokens.is_empty() {
            return Err(Error::from_reason("Empty key sequence".to_string()));
        }

        let mut modifiers = Vec::new();
        let mut main_key = None;

        for token in &tokens {
            let key = map_single_key(token)?;
            match key {
                enigo::Key::Alt | enigo::Key::Control | enigo::Key::Shift | enigo::Key::Meta => {
                    modifiers.push(key);
                }
                _ => {
                    if main_key.is_some() {
                        return Err(Error::from_reason("Multiple non-modifier keys in sequence".to_string()));
                    }
                    main_key = Some(key);
                }
            }
        }

        let main_key = main_key.ok_or_else(|| Error::from_reason("No main key specified".to_string()))?;
        Ok((modifiers, main_key))
    }

    /// 执行完整的组合键序列
    fn execute_key_sequence(&mut self, sequence: &str) -> Result<()> {
        let (modifiers, main_key) = self.parse_key_sequence(sequence)?;

        // 1. 按下所有修饰键
        for &mod_key in &modifiers {
            self.enigo.key(mod_key, enigo::Direction::Press)
                .map_err(|e| Error::from_reason(e.to_string()))?;
        }

        // 2. 点击主键（按下并释放）
        self.enigo.key(main_key, enigo::Direction::Click)
            .map_err(|e| Error::from_reason(e.to_string()))?;

        // 3. 释放修饰键（逆序释放）
        for &mod_key in modifiers.iter().rev() {
            self.enigo.key(mod_key, enigo::Direction::Release)
                .map_err(|e| Error::from_reason(e.to_string()))?;
        }

        Ok(())
    }
}

// === 单键映射表 ===
fn map_single_key(key: &str) -> Result<enigo::Key> {
    use enigo::Key;
    match key.to_lowercase().as_str() {
        // 单个字母动态处理
        s if s.len() == 1 && s.chars().next().unwrap().is_alphabetic() => {
            let c = s.chars().next().unwrap().to_ascii_uppercase();
            match c {
                'A' => Ok(Key::A), 'B' => Ok(Key::B), 'C' => Ok(Key::C),
                'D' => Ok(Key::D), 'E' => Ok(Key::E), 'F' => Ok(Key::F),
                'G' => Ok(Key::G), 'H' => Ok(Key::H), 'I' => Ok(Key::I),
                'J' => Ok(Key::J), 'K' => Ok(Key::K), 'L' => Ok(Key::L),
                'M' => Ok(Key::M), 'N' => Ok(Key::N), 'O' => Ok(Key::O),
                'P' => Ok(Key::P), 'Q' => Ok(Key::Q), 'R' => Ok(Key::R),
                'S' => Ok(Key::S), 'T' => Ok(Key::T), 'U' => Ok(Key::U),
                'V' => Ok(Key::V), 'W' => Ok(Key::W), 'X' => Ok(Key::X),
                'Y' => Ok(Key::Y), 'Z' => Ok(Key::Z),
                _ => Err(Error::from_reason(format!("Unsupported letter: {}", s))),
            }
        }
        "f1" => Ok(Key::F1), "f2" => Ok(Key::F2), "f3" => Ok(Key::F3),
        "f4" => Ok(Key::F4), "f5" => Ok(Key::F5), "f6" => Ok(Key::F6),
        "f7" => Ok(Key::F7), "f8" => Ok(Key::F8), "f9" => Ok(Key::F9),
        "f10" => Ok(Key::F10), "f11" => Ok(Key::F11), "f12" => Ok(Key::F12),
        "escape" | "esc" => Ok(Key::Escape),
        "enter" | "return" => Ok(Key::Return),
        "space" => Ok(Key::Space),
        "backspace" => Ok(Key::Backspace),
        "tab" => Ok(Key::Tab),
        "control" | "ctrl" => Ok(Key::Control),
        "alt" => Ok(Key::Alt),
        "shift" => Ok(Key::Shift),
        "meta" | "windows" | "command" => Ok(Key::Meta),
        "up" => Ok(Key::UpArrow),
        "down" => Ok(Key::DownArrow),
        "left" => Ok(Key::LeftArrow),
        "right" => Ok(Key::RightArrow),
        _ => Err(Error::from_reason(format!("Unsupported key: {}", key))),
    }
}

// === NAPI 暴露方法（替换原实现） ===
#[napi]
pub fn key(&mut self, sequence: String) -> Result<()> {
    self.execute_key_sequence(&sequence)
}
